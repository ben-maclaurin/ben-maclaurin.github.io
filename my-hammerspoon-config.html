My Hammerspoon config

Hammerspoon is a great tool for automating OS X tasks. In this short post, I'll share my configuration and its purpose.

I actually use two configs. One personal, one professional. However, I've mapped the hotkeys in a logical fashion to preserve my sanity. Currently, the configs serve two purposes (if you place window and application management under the same umbrella):

  • Window management
  • Browser actions

First, for window management, I've modelled the frame manipulation on Vim keybindings:

  • H = width / 2 and left align
  • J = height / 2 and bottom align
  • K = height / 2 and top align
  • L = width / 2 and right align

Finally, the F key, a little less synonymous, maximises the window. Here is an example: 

hs.hotkey.bind({"cmd", "shift"}, "H", function()
    local win = hs.window.focusedWindow()
    local f = win:frame()
    local screen = win:screen()
    local max = screen:frame()

    f.x = max.x
    f.h = max.h
    f.w = max.w / 2
    win:setFrame(f)
end)

Next comes application switching. For this I use the first two rows of the number pad: 7, 8, 9, 4, 5, 6. Row one houses the standard application shortcuts for both computers:

  • 7 = Firefox
  • 8 = Emacs
  • 9 = Finder

Now the second row, for custom applications:

Key Personal Work
4 Sublime Text PhpStorm
5 Alacritty Figma
6 Spotify Microsoft Teams

Even this row has some form of organisation. 4 is the primary focus (code), 5 is for secondary focus (i.e. checking logs or designs), and 6 for social or miscellaneous functions.

Finally, the last row, which consists of various browser actions:

  • 1 = open new empty tab
  • 2 = show browsing history
  • 3 = open http://localhost:3000/
