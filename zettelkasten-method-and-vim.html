Vim and Zettelkasten

Maintaining a repository of thoughts is not a new idea. A plethora of note-taking tools exist today, such as Obsidian, RemNote and Roam Research, all of which seem inspired by the Zettelkasten method. The technique, developed by the Swiss physician Conrad Gessner, helps its users collate personal knowledge via numbered cards and tags. How to Take Smart Notes provides great insight into the approach.

Whilst I don't doubt the usefulness of the aforementioned apps, they fail to engage me. This is because the simplicity of the Zettelkasten is what attracted me in the first place. There is a definite serenity to benefit from, and friction is eliminated because there are no abstractions, UIs or complex knowledge graphs to distract. It's just you, the pen and the paper.

The only app I came close to using is "The Archive". Unlike most other tools, it very clearly attributes its founding premise to the Zettelkasten, and its creators, Christian and Sascha, are also authors of https://zettelkasten.de, a site dedicated to the technique. If I weren't a programmer, this would unequivocally be my tool of choice, and I highly recommend you give its 60 day trial a shot.

After using "The Archive" for a couple of days, I couldn't help but wonder: can I replicate this process in Vim? I knew it was possible, but how much configuration would it take? It turns out just one line.

Why it works

What would this solution require to be effective? These were my requirements:

  • Basic editor (obviously)
  • Version control
  • Search
  • Unique note identifiers
  • Bi-directional linking
  • Dates and timestamps
  • Index files

Editor

If you haven't guessed, the editor is Vim. If you are a programmer, writer, or avid typist, I definitely suggest checking it out. It has dramatically improved my editing speed and workflow.

Version control

For version control I use Git. Nothing special or surprising here. In addition to version control, Git provides a backup of my notes, and a method of quick retrieval should I be using other machines.

Unique idenitifiers

You may have noticed the seemingly random set of numbers prefixing each filename. In reality, they are not random at all, and serve an important purpose. These numbers are Unix timestamps. Not only does this ensure every note is unique, it also provides a way of dating thoughts. For example, the id for the file 1642627149 Polygot computing.txt is 1642627149, which in GMT resolves to Wednesday, 19 January 2022 21:19:09.

Bi-directional linking

Jumping to another file path is possible by selecting it in visual mode and hitting gf. You can then navigate backwards using using CTRL + o.

Since the unix timestamps are difficult to recall from memory, here we find the single line of vimscript mentioned at the beginning of the post:

nnoremap cp :let @" = expand("%:t")<cr>

When cp is pressed, it conveniently copies the open filename to the clipboard. It can then be pasted into another note to create a bi-directional link:

Index files

A feature of Gessner's Zettelkasten are index files. If it is not clear from the name, index files act as entry points for particular topics. This is a good way to:

  1. Identify links between existing notes and new notes
  2. Retrieve all notes on a specific subject
